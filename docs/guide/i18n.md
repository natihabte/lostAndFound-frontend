# Multi-language Support (i18n)

The platform provides comprehensive internationalization support, allowing users to interact with the system in their preferred language.

## Supported Languages

The platform currently supports four languages:

1. **English** (en) - Default language
2. **Amharic** (አማርኛ) (am) - Ethiopian language
3. **Oromo** (Afaan Oromoo) (om) - Ethiopian language
4. **Tigrinya** (ትግርኛ) (ti) - Ethiopian/Eritrean language

## Language Selection

### First-Time Users

When you first visit the platform:
1. You'll see a language selector on the landing page
2. Choose your preferred language
3. The entire interface updates immediately
4. Your choice is saved for future visits

### Changing Language

**Header Language Selector:**
- Located in the top navigation bar
- Click the language button (e.g., "EN", "አማ", "OM", "ትግ")
- Select your preferred language
- Interface updates without page reload

**Profile Settings:**
- Navigate to Settings → Language
- Select preferred language
- Save changes
- Language persists across all sessions

## Technical Implementation

### For Developers

The platform uses **i18next** for internationalization:

```javascript
import { useTranslation } from 'react-i18next';

function MyComponent() {
  const { t, i18n } = useTranslation();

  return (
    <div>
      <h1>{t('welcome.title')}</h1>
      <p>{t('welcome.description')}</p>
      <button onClick={() => i18n.changeLanguage('am')}>
        {t('language.amharic')}
      </button>
    </div>
  );
}
```

### Translation Files

Located in `src/i18n/locales/`:

```
src/i18n/locales/
├── en.json    # English translations
├── am.json    # Amharic translations
├── om.json    # Oromo translations
└── ti.json    # Tigrinya translations
```

### Translation Keys Structure

```json
{
  "common": {
    "welcome": "Welcome",
    "login": "Login",
    "logout": "Logout"
  },
  "items": {
    "lost": "Lost Items",

  },
  "categories": {
    "electronics": "Electronics",

}
```

## Language-Specific Features

### Date and Time Formatting

Dates and times are formatted according to the selected language:

- **English**: January 15, 2024, 3:30 PM
- **Amharic**: ጃንዋሪ 15, 2024, 3:30 ከሰዓት
- **Oromo**: Amajjii 15, 2024, 3:30 WB
- **Tigrinya**: ጥሪ 15, 2024, 3:30 ድሕሪ ቀትሪ

### Number Formatting

Numbers are formatted according to locale conventions:
- Decimal separators
- Thousand separators
- Currency formatting

### Pluralization

The system handles plural forms correctly for each language:

```javascript
// English: 1 item, 2 items
// Amharic: 1 ዕቃ, 2 ዕቃዎች
t('items.count', { count: itemCount })
```

## Adding New Languages

### For Administrators

To add a new language:

1. **Create Translation File**
   - Copy `en.json` as template
   - Translate all keys
   - Save as `[language-code].json`

2. **Update Configuration**
   ```javascript
   // src/i18n/index.js
   i18n.use(initReactI18next).init({
     resources: {
       en: { translation: enTranslations },
       am: { translation: amTranslations },
       om: { translation: omTranslations },
       ti: { translation: tiTranslations },
       // Add new language here
       sw: { translation: swTranslations }
     }
   });
   ```

3. **Add Language Selector Option**
   ```javascript
   const languages = [
     { code: 'en', name: 'English', nativeName: 'English' },
     { code: 'am', name: 'Amharic', nativeName: 'አማርኛ' },
     { code: 'om', name: 'Oromo', nativeName: 'Afaan Oromoo' },
     { code: 'ti', name: 'Tigrinya', nativeName: 'ትግርኛ' },
     // Add new language
     { code: 'sw', name: 'Swahili', nativeName: 'Kiswahili' }
   ];
   ```

## Translation Guidelines

### For Translators

When translating content:

1. **Maintain Context**: Understand the context of each string
2. **Be Consistent**: Use consistent terminology throughout
3. **Keep Length Similar**: Try to match original text length
4. **Preserve Placeholders**: Don't translate `{{variable}}` placeholders
5. **Test Thoroughly**: Verify translations in the actual interface

### Translation Keys Best Practices

```json
{
  // Good: Descriptive, hierarchical
  "items.lost.report.title": "Report Lost Item",
  
  // Bad: Vague, flat structure
  "title1": "Report Lost Item",
  
  // Good: Includes context
  "button.submit.form": "Submit",
  
  // Bad: No context
  "submit": "Submit"
}
```

## Troubleshooting

### Language Not Changing

1. **Clear browser cache**: Hard refresh (Ctrl+Shift+R)
2. **Check browser settings**: Ensure JavaScript is enabled
3. **Verify account settings**: Check profile language preference
4. **Try different browser**: Rule out browser-specific issues

### Missing Translations

If you see untranslated text:
1. **Report to admin**: Use feedback form
2. **Provide context**: Screenshot and location
3. **Suggest translation**: If you know the language
4. **Use fallback**: System falls back to English

### Text Overflow Issues

If translated text doesn't fit:
1. **Report the issue**: Include language and location
2. **Temporary workaround**: Switch to English
3. **Admin will fix**: Layout will be adjusted
